## Εισαγωγή

Από την έκδοση **5.0.1-alpha**, το PyArchInit ενσωματώνει ένα σύστημα **Καθολικά Μοναδικών Αναγνωριστικών (UUID)** για όλες τις αρχαιολογικές οντότητες. Αυτή η λειτουργικότητα αποτελεί μέρος του ευρωπαϊκού προγράμματος **StratiGraph** (Horizon Europe) και διασφαλίζει ότι κάθε εγγραφή στη βάση δεδομένων διαθέτει ένα σταθερό και παγκοσμίως μοναδικό αναγνωριστικό.

## Τι είναι τα UUID

Ένα **UUID** (Universally Unique Identifier) είναι ένας αλφαριθμητικός κώδικας 128 bit που αναγνωρίζει μοναδικά μια οντότητα. Το PyArchInit χρησιμοποιεί την έκδοση 4 (UUID v4), η οποία δημιουργείται τυχαία.

### Παράδειγμα UUID

```
a3f7b2c1-8d4e-4f5a-9b6c-1234567890ab
```

### Χαρακτηριστικά UUID

| Χαρακτηριστικό | Περιγραφή |
|---------------|-----------|
| **Μορφή** | 32 δεκαεξαδικοί χαρακτήρες διαχωρισμένοι με παύλες (8-4-4-4-12) |
| **Μοναδικότητα** | Η πιθανότητα σύγκρουσης είναι στατιστικά αμελητέα (~1 στα 2^122) |
| **Ανεξαρτησία** | Δεν εξαρτάται από τη βάση δεδομένων, τον διακομιστή ή τον χρόνο δημιουργίας |
| **Μονιμότητα** | Μόλις αντιστοιχηθεί, δεν αλλάζει ποτέ |
| **Έκδοση** | UUID v4 (τυχαία δημιουργία) |

### Διαφορά με τα παραδοσιακά αναγνωριστικά

| Τύπος ID | Παράδειγμα | Σταθερό μεταξύ ΒΔ; | Παγκοσμίως μοναδικό; |
|----------|---------|-------------------|---------------------|
| Αυτόματη αύξηση (id_us) | `1`, `2`, `3`... | Όχι | Όχι |
| Σύνθετος περιορισμός | `Site1-Area1-US100` | Ναι (σημασιολογικά) | Εξαρτάται |
| **UUID** | `a3f7b2c1-8d4e-...` | **Ναι** | **Ναι** |

Τα αυτο-αυξανόμενα αναγνωριστικά (`id_us`, `id_invmat` κ.λπ.) αλλάζουν όταν αντιγράφετε μια βάση δεδομένων ή εισάγετε δεδομένα. Τα UUID, αντίθετα, παραμένουν **πάντα τα ίδια**, ανεξάρτητα από το πού βρίσκονται τα δεδομένα.

## Γιατί χρειάζονται τα UUID στο StratiGraph

Το πρόγραμμα StratiGraph απαιτεί τα αρχαιολογικά δεδομένα να μπορούν να:

### 1. Εξαχθούν στο Knowledge Graph

Τα δεδομένα του PyArchInit εξάγονται ως **πακέτα** (δομημένα πακέτα δεδομένων) σε ένα κεντρικό Knowledge Graph. Κάθε οντότητα πρέπει να έχει ένα σταθερό αναγνωριστικό για να αναγνωρίζεται στον γράφο.

```
Τοπική οντότητα (PyArchInit)  -->  UUID  -->  Knowledge Graph (StratiGraph)
     SU 100                   a3f7b2c1...        E18 Physical Thing
```

### 2. Συγχρονιστούν μεταξύ συσκευών

Όταν εργάζεστε στο πεδίο χωρίς σύνδεση στο διαδίκτυο, τα δεδομένα αποθηκεύονται τοπικά. Κατά την αποκατάσταση της συνδεσιμότητας, τα δεδομένα συγχρονίζονται. Τα UUID διασφαλίζουν ότι η ίδια εγγραφή αναγνωρίζεται και ενημερώνεται (δεν δημιουργείται αντίγραφο).

### 3. Αντιστοιχηθούν στο CIDOC-CRM

Η οντολογία CIDOC-CRM απαιτεί **μόνιμα URI** για κάθε οντότητα. Τα UUID χρησιμοποιούνται για την κατασκευή αυτών των URI:

```
http://pyarchinit.org/entity/a3f7b2c1-8d4e-4f5a-9b6c-1234567890ab
```

### 4. Παρακολουθηθούν σε βάθος χρόνου

Κάθε τροποποίηση, εξαγωγή ή συγχρονισμός αναφέρεται στο ίδιο UUID. Αυτό επιτρέπει:
- Ανασύνθεση του ιστορικού μιας εγγραφής
- Επαλήθευση της προέλευσης δεδομένων
- Σύνδεση δεδομένων μεταξύ διαφορετικών έργων

## Πώς λειτουργούν στο PyArchInit

### Αυτόματη δημιουργία

Τα UUID δημιουργούνται **αυτόματα** σε δύο χρονικές στιγμές:

| Στιγμή | Περιγραφή |
|--------|-----------|
| **Δημιουργία νέας εγγραφής** | Κατά την εισαγωγή νέας εγγραφής (π.χ. νέα ΣΜ), δημιουργείται αυτόματα ένα UUID v4 |
| **Μετανάστευση υπάρχουσας βάσης** | Κατά την πρώτη εκκίνηση μετά την ενημέρωση, όλες οι υπάρχουσες εγγραφές χωρίς UUID λαμβάνουν δημιουργημένο UUID |

Ο χρήστης **δεν χρειάζεται να κάνει τίποτα**: τα UUID διαχειρίζονται εξ ολοκλήρου από το σύστημα.

### Πού βρίσκεται το UUID

Κάθε κύριος πίνακας της βάσης δεδομένων διαθέτει μια στήλη `entity_uuid` τύπου TEXT. Το πεδίο είναι ορατό στη βάση δεδομένων αλλά δεν εμφανίζεται στις φόρμες εισαγωγής δεδομένων, καθώς διαχειρίζεται εσωτερικά.

### Αυτόματη μετανάστευση

Κατά την ενημέρωση του PyArchInit στην έκδοση 5.0.1-alpha (ή μεταγενέστερη):

1. **Κατά την πρώτη εκκίνηση**, το σύστημα ελέγχει εάν οι πίνακες διαθέτουν τη στήλη `entity_uuid`
2. Εάν λείπει, η στήλη **προστίθεται αυτόματα**
3. Οι υπάρχουσες εγγραφές χωρίς UUID λαμβάνουν **δημιουργημένο UUID**
4. Αυτή η λειτουργία πραγματοποιείται **μόνο μία φορά** ανά συνεδρία QGIS

Η διαδικασία είναι διαφανής και δεν απαιτεί χειροκίνητη παρέμβαση. Λειτουργεί τόσο με **PostgreSQL** όσο και με **SQLite**.

## Συχνές Ερωτήσεις

### Χρειάζεται να εισάγω τα UUID χειροκίνητα;

**Όχι.** Τα UUID δημιουργούνται αυτόματα από το σύστημα. Δεν είναι αναγκαίο (ούτε συνιστάται) να τα τροποποιείτε χειροκίνητα.

### Τι συμβαίνει εάν αντιγράψω τη βάση δεδομένων;

Τα UUID αντιγράφονται μαζί με τη βάση δεδομένων. Αυτή είναι η επιθυμητή συμπεριφορά: η ίδια εγγραφή διατηρεί το ίδιο UUID ακόμη και σε διαφορετικά αντίγραφα της βάσης δεδομένων.

### Μπορώ να δω τα UUID στις φόρμες;

Προς το παρόν, τα UUID δεν είναι ορατά στις φόρμες εισαγωγής δεδομένων. Είναι ορατά απευθείας στη βάση δεδομένων (π.χ. μέσω DB Manager στο QGIS) στη στήλη `entity_uuid` κάθε πίνακα.

### Τα UUID επιβραδύνουν τη βάση δεδομένων;

Όχι. Το UUID είναι ένα απλό πεδίο TEXT και δεν έχει σημαντικό αντίκτυπο στην απόδοση της βάσης δεδομένων.

### Τι συμβαίνει με τις υπάρχουσες βάσεις δεδομένων;

Οι υπάρχουσες βάσεις δεδομένων ενημερώνονται αυτόματα κατά την πρώτη εκκίνηση: η στήλη `entity_uuid` προστίθεται και όλες οι υπάρχουσες εγγραφές λαμβάνουν δημιουργημένο UUID.

### Τα UUID λειτουργούν τόσο με PostgreSQL όσο και με SQLite;

Ναι. Το σύστημα είναι συμβατό και με τους δύο τύπους βάσεων δεδομένων που υποστηρίζονται από το PyArchInit.
