# Εκπαιδευτικό 05: Δελτίο Κατασκευής

## Εισαγωγή

Το **Δελτίο Κατασκευής** είναι η ενότητα του PyArchInit αφιερωμένη στην τεκμηρίωση αρχαιολογικών κατασκευών. Μια κατασκευή είναι ένα οργανωμένο σύνολο Στρωματογραφικών Μονάδων (ΣΜ/ΤΣΜ) που σχηματίζουν μια αναγνωρίσιμη κατασκευαστική ή λειτουργική οντότητα, όπως τοίχος, δάπεδο, τάφος, κλίβανος ή οποιοδήποτε άλλο δομημένο στοιχείο.

### Βασικές Έννοιες

**Κατασκευή έναντι Στρωματογραφικής Μονάδας:**
- Μια **ΣΜ** είναι η μεμονωμένη μονάδα (στρώμα, τομή, πλήρωση)
- Μια **Κατασκευή** ομαδοποιεί πολλαπλές λειτουργικά σχετιζόμενες ΣΜ
- Παράδειγμα: ένας τοίχος (κατασκευή) αποτελείται από θεμελίωση, ανωδομή, κονίαμα (διαφορετικές ΣΜ)

**Ιεραρχίες:**
- Οι κατασκευές μπορούν να έχουν σχέσεις μεταξύ τους
- Κάθε κατασκευή ανήκει σε μία ή περισσότερες χρονολογικές περιόδους/φάσεις
- Οι κατασκευές συνδέονται με τις ΣΜ που τις αποτελούν

## Πεδία Ταυτοποίησης

Τα πεδία ταυτοποίησης ορίζουν μοναδικά την κατασκευή στη βάση δεδομένων.

![Πεδία ταυτοποίησης](images/05_scheda_struttura/06_campi_identificativi.png)

### Θέση

**Πεδίο**: `comboBox_sito`
**Βάση δεδομένων**: `sito`

Επιλέξτε την αρχαιολογική θέση στην οποία ανήκει. Το πτυσσόμενο μενού εμφανίζει όλες τις θέσεις καταχωρημένες στη βάση δεδομένων.

**Σημειώσεις:**
- Υποχρεωτικό πεδίο
- Ο συνδυασμός Θέση + Κωδικός + Αριθμός πρέπει να είναι μοναδικός
- Κλειδώνεται μετά τη δημιουργία εγγραφής

### Κωδικός Κατασκευής

**Πεδίο**: `comboBox_sigla_struttura`
**Βάση δεδομένων**: `sigla_struttura`

Αλφαριθμητικός κωδικός που ταυτοποιεί τον τύπο κατασκευής. Συνηθισμένες συμβάσεις:

| Κωδικός | Σημασία | Παράδειγμα |
|---------|---------|------------|
| WL | Τοίχος | WL 1 |
| ST | Κατασκευή | ST 5 |
| FL | Δάπεδο | FL 2 |
| KN | Κλίβανος | KN 1 |
| TK | Δεξαμενή | TK 3 |
| TB | Τάφος | TB 10 |
| WE | Πηγάδι | WE 2 |
| CN | Κανάλι | CN 1 |

**Σημειώσεις:**
- Επεξεργάσιμο με ελεύθερη εισαγωγή
- Κλειδώνεται μετά τη δημιουργία

### Αριθμός Κατασκευής

**Πεδίο**: `numero_struttura`
**Βάση δεδομένων**: `numero_struttura`

Προοδευτικός αριθμός κατασκευής εντός του κωδικού.

**Σημειώσεις:**
- Αριθμητικό πεδίο
- Πρέπει να είναι μοναδικός για τον συνδυασμό Θέση + Κωδικός
- Κλειδώνεται μετά τη δημιουργία

## Συγχρονισμός με Δελτίο ΣΜ

Οι κατασκευές που δημιουργούνται σε αυτό το δελτίο εμφανίζονται αυτόματα στο πεδίο **Structure** του **Δελτίου ΣΜ** για την ίδια θέση.

### Πώς λειτουργεί η σύνδεση

1. **Δημιουργήστε την κατασκευή** συμπληρώνοντας τουλάχιστον:
   - **Site**: η αρχαιολογική θέση (π.χ., "Rome_Forum")
   - **Code**: τον κωδικό κατασκευής (π.χ., "MUR")
   - **Number**: τον προοδευτικό αριθμό (π.χ., 1)
   - Αποθηκεύστε την εγγραφή

2. **Στο Δελτίο ΣΜ** για την ίδια θέση:
   - Το πεδίο **Structure** θα εμφανίζει την κατασκευή σε μορφή `CODE-NUMBER`
   - Παράδειγμα: "MUR-1", "PV-2", "ST-3"

### Μορφή εμφάνισης

Οι κατασκευές εμφανίζονται στο Δελτίο ΣΜ σε μορφή:
```
ΚΩΔΙΚΟΣ_ΚΑΤΑΣΚΕΥΗΣ - ΑΡΙΘΜΟΣ_ΚΑΤΑΣΚΕΥΗΣ
```

**Παραδείγματα:**
| Δελτίο Κατασκευής | Εμφάνιση στο Δελτίο ΣΜ |
|-------------------|------------------------|
| Site=Rome, Code=MUR, Number=1 | MUR-1 |
| Site=Rome, Code=PV, Number=2 | PV-2 |
| Site=Rome, Code=ST, Number=10 | ST-10 |

### Σημαντικές σημειώσεις

- Η κατασκευή πρέπει να είναι **αποθηκευμένη** πριν εμφανιστεί στο Δελτίο ΣΜ
- Μόνο κατασκευές από την **ίδια θέση** είναι ορατές
- Στο Δελτίο ΣΜ μπορείτε να **επιλέξετε πολλαπλές κατασκευές** (πολλαπλή επιλογή με πλαίσια ελέγχου)
- Για **αφαίρεση** κατασκευής από τη ΣΜ: ανοίξτε το πτυσσόμενο μενού και αποεπιλέξτε
- Για **εκκαθάριση** όλων των κατασκευών: δεξί κλικ → "Clear Structure field"

**Πεδίο**: `textEdit_descrizione_struttura`
**Βάση δεδομένων**: `descrizione`

Πεδίο ελεύθερου κειμένου για φυσική περιγραφή της κατασκευής.

**Προτεινόμενο περιεχόμενο:**
- Οικοδομική τεχνική
- Χρησιμοποιούμενα υλικά
- Κατάσταση διατήρησης
- Γενικές διαστάσεις
- Προσανατολισμός
- Ιδιαίτερα χαρακτηριστικά

**Παράδειγμα:**
```
Τοίχος σε opus incertum κατασκευασμένος με τοπικούς ασβεστολιθικούς
ογκολίθους μεταβαλλόμενου μεγέθους (15-30 εκ.). Λευκωπό κονίαμα
ασβέστη ως συνδετικό υλικό. Διατηρείται σε μέγιστο ύψος 1,20 μ.
Μέσο πλάτος 50 εκ. Προσανατολισμός ΒΑ-ΝΔ. Εμφανίζει ίχνη
επιχρίσματος στην εσωτερική πλευρά.
```

**Πεδίο**: `textEdit_interpretazione_struttura`

Λειτουργική και ιστορική ερμηνεία της κατασκευής.

**Προτεινόμενο περιεχόμενο:**
- Υποτιθέμενη αρχική λειτουργία
- Φάσεις χρήσης/επαναχρησιμοποίησης
- Τυπολογικές συγκρίσεις
- Χρονολογικό πλαίσιο
- Σχέσεις με άλλες κατασκευές

## Ροή Εργασίας

### Δημιουργία Νέας Κατασκευής

1. **Άνοιγμα δελτίου** - Μέσω μενού ή γραμμής εργαλείων

2. **Νέα εγγραφή** - Κάντε κλικ στο "New Record"

3. **Δεδομένα ταυτοποίησης**
   ```
   Site: Ρωμαϊκή Έπαυλη Settefinestre
   Code: WL
   Number: 15
   ```

4. **Κατάταξη**
   ```
   Category: Οικιστική
   Type: Περιμετρικός τοίχος
   Definition: Τοίχος opus incertum
   ```

5. **Περιγραφικά δεδομένα** (Καρτέλα 1)

6. **Περιοδολόγηση** (Καρτέλα 2)

7. **Σχέσεις** (Καρτέλα 3)

8. **Δομικά στοιχεία** (Καρτέλα 4)

9. **Μετρήσεις** (Καρτέλα 5)

10. **Αποθήκευση** - Κάντε κλικ στο "Save"

## Βέλτιστες Πρακτικές

### Ονοματοδοσία
- Χρησιμοποιήστε συνεπείς κωδικούς σε όλο το έργο
- Τεκμηριώστε τις συμβάσεις που υιοθετήθηκαν
- Αποφύγετε τη διπλή αρίθμηση

### Περιγραφή
- Ακολουθήστε ένα σχήμα: τεχνική > υλικά > διαστάσεις > κατάσταση
- Διαχωρίστε αντικειμενική περιγραφή από ερμηνεία

### Περιοδολόγηση

## Αντιμετώπιση Προβλημάτων

### Πρόβλημα: Η κατασκευή δεν είναι ορατή στον χάρτη
**Λύση**: Επαληθεύστε ότι υπάρχει γεωμετρία και το επίπεδο `pyarchinit_strutture` είναι φορτωμένο

### Πρόβλημα: Οι περίοδοι δεν είναι διαθέσιμες
**Λύση**: Ανοίξτε πρώτα το Δελτίο Περιοδολόγησης και ορίστε περιόδους

### Πρόβλημα: Σφάλμα αποθήκευσης "existing record"
**Λύση**: Ο συνδυασμός Θέση + Κωδικός + Αριθμός υπάρχει ήδη
