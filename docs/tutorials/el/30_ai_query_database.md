# Οδηγός 30: Ερώτημα Βάσης Δεδομένων με ΤΝ (Text2SQL)

## Εισαγωγή

Η λειτουργία **AI Query Database** είναι ένα προηγμένο χαρακτηριστικό του PyArchInit που επιτρέπει την υποβολή ερωτημάτων στην αρχαιολογική βάση δεδομένων με χρήση **φυσικής γλώσσας**. Το σύστημα μετατρέπει αυτόματα τις ερωτήσεις σε ερωτήματα SQL χρησιμοποιώντας τεχνητή νοημοσύνη.

### Πώς Λειτουργεί

1. Ο χρήστης γράφει μια ερώτηση στα Αγγλικά/Ιταλικά
2. Η ΤΝ αναλύει το αίτημα
3. Δημιουργεί το αντίστοιχο ερώτημα SQL
4. Εκτελεί το ερώτημα στη βάση δεδομένων
5. Επιστρέφει τα αποτελέσματα

### Παραδείγματα Ερωτήσεων

- *"Find all ceramic finds from the Roman Villa site"*
- *"Show Roman period SUs in area 1000"*
- *"How many individuals were found in the graves?"*
- *"List structures with medieval dating"*

## Πρόσβαση

### Από τη Φόρμα ΣΜ/Ευρημάτων
Ειδική καρτέλα **"AI Query"** ή **"Text2SQL"**

### Από το Μενού
**PyArchInit** → **AI Query Database**

## Λειτουργίες Δημιουργίας

### 1. OpenAI GPT-4

- **Απαιτήσεις**: Ρυθμισμένο κλειδί API OpenAI
- **Ποιότητα**: Εξαιρετική
- **Κόστος**: Πληρωμή ανά χρήση (API)
- **Ταχύτητα**: Γρήγορη

### 2. Ollama (Τοπικό)

- **Απαιτήσεις**: Εγκατεστημένο και σε λειτουργία το Ollama
- **Ποιότητα**: Καλή-Εξαιρετική (εξαρτάται από το μοντέλο)
- **Κόστος**: Δωρεάν
- **Ταχύτητα**: Εξαρτάται από το υλικό

### 3. Free API

- **Απαιτήσεις**: Σύνδεση στο διαδίκτυο
- **Ποιότητα**: Μεταβλητή
- **Κόστος**: Δωρεάν
- **Περιορισμοί**: Πιθανός περιορισμός ρυθμού

## Ρύθμιση

### OpenAI

1. Αποκτήστε κλειδί API από το [OpenAI](https://platform.openai.com/)
2. Ρυθμίστε στο **PyArchInit** → **Configuration** → **API Keys**
3. Επιλέξτε "OpenAI GPT-4" στη λειτουργία

### Ollama

1. Εγκαταστήστε το Ollama από [ollama.ai](https://ollama.ai/)
2. Εκκινήστε το Ollama: `ollama serve`
3. Κατεβάστε μοντέλο: `ollama pull llama3.2`
4. Επιλέξτε "Ollama" και επαληθεύστε τη σύνδεση

### Συνιστώμενα Μοντέλα Ollama

| Μοντέλο | Μέγεθος | Ποιότητα SQL |
|---------|---------|-------------|
| llama3.2 | 2GB | Καλή |
| mistral | 4GB | Εξαιρετική |
| codellama | 7GB | Εξαιρετική για SQL |
| qwen2.5-coder | 4GB | Εξαιρετική για κώδικα |

## Χρήση

### 1. Γράψτε την Ερώτηση

Στο πλαίσιο εισόδου, περιγράψτε τι θέλετε να αναζητήσετε:
- Χρησιμοποιήστε φυσική γλώσσα
- Να είστε συγκεκριμένοι όταν είναι δυνατόν
- Αναφέρετε πίνακες/πεδία εάν τα γνωρίζετε

### 2. Δημιουργία SQL

1. Κάντε κλικ στο **"Generate SQL"**
2. Περιμένετε την επεξεργασία
3. Δείτε το δημιουργημένο ερώτημα

### 3. Επαλήθευση Ερωτήματος

- Διαβάστε το δημιουργημένο ερώτημα SQL
- Κάντε κλικ στο **"Explain Query"** για κατανόηση
- Επαληθεύστε τη λογική ορθότητα

### 4. Εκτέλεση Ερωτήματος

- **"Copy Query"**: Αντιγραφή στο πρόχειρο
- **"Use Query"**: Άμεση εκτέλεση στο σύστημα

## Σχήμα Βάσης Δεδομένων

### Κύριοι Πίνακες

Το σύστημα γνωρίζει το σχήμα του PyArchInit:

| Πίνακας | Περιγραφή |
|---------|-----------|
| site_table | Αρχαιολογικές θέσεις |
| us_table | Στρωματογραφικές Μονάδες |
| inventario_materiali_table | Ευρήματα |
| pottery_table | Κεραμική |
| tomba_table | Τάφοι |
| individui_table | Άτομα |
| struttura_table | Κατασκευές |
| periodizzazione_table | Χρονολόγηση |
| campioni_table | Δείγματα |
| documentazione_table | Τεκμηρίωση |

### Κοινά Πεδία

Η ΤΝ γνωρίζει τα κύρια πεδία:
- `sito` - Όνομα θέσης
- `area` - Αριθμός τομέα
- `us` - Αριθμός ΣΜ
- `periodo_iniziale` / `fase_iniziale`
- `datazione_estesa`
- `descrizione` / `interpretazione`

## Παραδείγματα Ερωτημάτων

### Αναζήτηση Ευρημάτων

**Ερώτηση**: *"Find all bronze finds from the Ancient Rome site"*

```sql
SELECT * FROM inventario_materiali_table
WHERE sito = 'Ancient Rome'
AND (tipo_reperto LIKE '%bronze%' OR definizione LIKE '%bronze%')
```

### Καταμέτρηση ΣΜ

**Ερώτηση**: *"How many SUs are there per period at Villa Adriana site?"*

```sql
SELECT periodo_iniziale, COUNT(*) as num_su
FROM us_table
WHERE sito = 'Villa Adriana'
GROUP BY periodo_iniziale
```

### Χωρική Αναζήτηση

**Ερώτηση**: *"Show SUs in area 1000 with elevations below 10 meters"*

```sql
SELECT * FROM us_table
WHERE area = '1000'
AND quota_min_usm < 10
```

## Βέλτιστες Πρακτικές

### 1. Αποτελεσματικές Ερωτήσεις

- Να είστε συγκεκριμένοι: ονόματα θέσεων, αριθμοί, ημερομηνίες
- Υποδείξτε τι θέλετε: λίστα, καταμέτρηση, λεπτομέρειες
- Αναφέρετε τα επιθυμητά φίλτρα

### 2. Επαλήθευση Αποτελεσμάτων

- Πάντα ελέγχετε το δημιουργημένο ερώτημα
- Χρησιμοποιήστε "Explain Query" εάν δεν είναι σαφές
- Δοκιμάστε σε υποσύνολο πριν από σύνθετα ερωτήματα

### 3. Επανάληψη

- Εάν το αποτέλεσμα είναι λανθασμένο, αναδιατυπώστε την ερώτηση
- Προσθέστε λεπτομέρειες εάν το ερώτημα είναι πολύ γενικό
- Απλοποιήστε εάν το ερώτημα είναι πολύ σύνθετο

## Αντιμετώπιση Προβλημάτων

### Δεν Δημιουργείται Ερώτημα

**Αιτίες**:
- Μη ρυθμισμένο API
- Έλλειψη σύνδεσης
- Μη κατανοητή ερώτηση

**Λύσεις**:
- Επαληθεύστε τη ρύθμιση API
- Ελέγξτε τη σύνδεση
- Αναδιατυπώστε την ερώτηση

### Λανθασμένα Αποτελέσματα

**Αιτίες**:
- Ασαφής ερώτηση
- Ανύπαρκτο πεδίο/πίνακας

**Λύσεις**:
- Να είστε πιο συγκεκριμένοι
- Επαληθεύστε ονόματα πινάκων/πεδίων

### Το Ollama Δεν Ανταποκρίνεται

**Αιτίες**:
- Το Ollama δεν εκτελείται
- Δεν έχει γίνει λήψη μοντέλου

**Λύσεις**:
- Εκκινήστε `ollama serve`
- Κατεβάστε το απαιτούμενο μοντέλο
