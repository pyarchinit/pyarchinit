## Εισαγωγη

Το **MoveCost** ειναι ενα αυτοτελες εργαλειο του PyArchInit για την αναλυση διαδρομων ελαχιστου κοστους (Least-Cost Path Analysis, LCPA) βασισμενο σε scripts R. Η αναλυση διαδρομων ελαχιστου κοστους ειναι μια θεμελιωδης μεθοδολογια στην αρχαιολογια τοπιου που επιτρεπει τη μοντελοποιηση των πιο πιθανων διαδρομων μεταξυ θεσεων, λαμβανοντας υποψη την τοπογραφια του εδαφους και το ενεργειακο κοστος της κινησης.

### Ιστορια

Προηγουμενως, η λειτουργικοτητα MoveCost ηταν ενσωματωμενη απευθειας στη φορμα Θεσης (Site form) του PyArchInit. Απο την τρεχουσα εκδοση, το MoveCost εξηχθη ως **ανεξαρτητο εργαλειο αναλυσης**, προσβασιμο μεσω ενος αφιερωμενου QDialog. Αυτος ο διαχωρισμος προσφερει πολλα πλεονεκτηματα:

- **Αφιερωμενη διεπαφη**: Ενας διαλογος με 4 καρτελες οργανωμενες κατα λειτουργια
- **Καλυτερη οργανωση**: Αλγοριθμοι, αποτελεσματα, εξαγωγη και ρυθμισεις σαφως διαχωρισμενα
- **Γρηγορη προσβαση**: Διαθεσιμο απο τη γραμμη εργαλειων χωρις να ανοιξετε τη φορμα Θεσης
- **Επεκτασιμοτητα**: Αρθρωτη δομη που διευκολυνει την προσθηκη νεων αλγοριθμων

### Τι ειναι η αναλυση διαδρομων ελαχιστου κοστους;

Η αναλυση διαδρομων ελαχιστου κοστους υπολογιζει τη βελτιστη διαδρομη μεταξυ δυο η περισσοτερων σημειων σε μια επιφανεια κοστους που προερχεται απο ενα ψηφιακο μοντελο εδαφους (DTM). Το κοστος της κινησης εξαρταται απο την κλιση του εδαφους και υπολογιζεται χρησιμοποιωντας ανισοτροπες συναρτησεις κοστους που λαμβανουν υποψη την κατευθυνση της κινησης (ανοδος εναντι καθοδου).

## Προαπαιτουμενα

Πριν χρησιμοποιησετε το MoveCost, βεβαιωθειτε οτι τα ακολουθα συστατικα ειναι εγκατεστημενα και ρυθμισμενα:

### 1. R (Στατιστικη γλωσσα)

| Απαιτηση | Λεπτομερεια |
|----------|------------|
| **Λογισμικο** | R (εκδοση >= 4.0 συνισταται) |
| **Ληψη** | [https://cran.r-project.org/](https://cran.r-project.org/) |
| **Επιβεβαιωση** | Ανοιξτε ενα τερματικο και πληκτρολογηστε `R --version` |

### 2. Πακετο R `movecost`

Εγκαταστηστε το πακετο μεσα απο το R:

```r
install.packages("movecost")
```

Κυριες εξαρτησεις εγκαθιστανται αυτοματα: `terra`, `gdistance`, `sp`.

### 3. QGIS Processing R Provider

| Απαιτηση | Λεπτομερεια |
|----------|------------|
| **Προσθετο** | Processing R Provider |
| **Εγκατασταση** | QGIS > Προσθετα > Διαχειριση και εγκατασταση προσθετων > Αναζητηση "Processing R Provider" |
| **Ρυθμιση** | Ρυθμισεις επεξεργασιας > Παροχοι > R > Διαδρομη φακελου R |

### 4. Δεδομενα εισοδου

- **DTM/DEM**: Ενα ψηφιακο μοντελο εδαφους raster για την περιοχη μελετης
- **Επιπεδο σημειων**: Σημεια προελευσης και προορισμου για την αναλυση
- **Επιπεδο πολυγωνων**: (Προαιρετικο) Για τις παραλλαγες "by polygon" των αλγοριθμων

### Γρηγορη λιστα ελεγχου προαπαιτουμενων

```
+-------------------------------------------+
| Λιστα ελεγχου προαπαιτουμενων              |
+-------------------------------------------+
| [x] R εγκατεστημενο και στο PATH         |
| [x] Πακετο movecost εγκατεστημενο στο R  |
| [x] Processing R Provider ενεργο στο QGIS|
| [x] DTM φορτωμενο στο εργο QGIS           |
| [x] Επιπεδο σημειων με προελευσεις/προορ. |
+-------------------------------------------+
```

## Καρτελα Αλγοριθμων

Η καρτελα **Αλγοριθμοι** ειναι ο πυρηνας του εργαλειου MoveCost. Περιεχει **14 αλγοριθμους** βασισμενους σε scripts R, οργανωμενους σε **3 λειτουργικες ομαδες**.

### Ομαδα 1: Επιφανεια κοστους και διαδρομες

Αλγοριθμοι για τον υπολογισμο σωρευτικων επιφανειων κοστους και διαδρομων ελαχιστου κοστους.

| Αλγοριθμος | Περιγραφη |
|------------|-----------|
| **movecost** | Υπολογιζει τη σωρευτικη ανισοτροπη επιφανεια κοστους εξαρτωμενη απο την κλιση και τις διαδρομες ελαχιστου κοστους απο ενα σημειο προελευσης |
| **movecost by polygon** | Το ιδιο αλλα χρησιμοποιωντας μια πολυγωνικη περιοχη για τον ορισμο της εκτασης του DTM |
| **movebound** | Υπολογιζει τα ορια κοστους μετακινησης εξαρτωμενα απο την κλιση γυρω απο σημειακες θεσεις |
| **movebound by polygon** | Το ιδιο αλλα χρησιμοποιωντας ενα πολυγωνο |

### Ομαδα 2: Αναλυση διαδρομων και δικτυων

Αλγοριθμοι για την αναλυση διαδρομων κοστους και δικτυων βελτιστων διαδρομων.

| Αλγοριθμος | Περιγραφη |
|------------|-----------|
| **movecorr** | Υπολογιζει τον διαδρομο ελαχιστου κοστους μεταξυ σημειακων θεσεων |
| **movecorr by polygon** | Το ιδιο αλλα χρησιμοποιωντας ενα πολυγωνο |
| **movealloc** | Υπολογιζει την κατανομη κοστους μετακινησης εξαρτωμενη απο την κλιση στις προελευσεις |
| **movealloc by polygon** | Το ιδιο αλλα χρησιμοποιωντας ενα πολυγωνο |
| **movenetw** | Υπολογιζει το δικτυο διαδρομων ελαχιστου κοστους μεταξυ πολλαπλων σημειων |
| **movenetw by polygon** | Το ιδιο αλλα χρησιμοποιωντας ενα πολυγωνο |

### Ομαδα 3: Συγκριση και καταταξη

Αλγοριθμοι για τη συγκριση συναρτησεων κοστους και την καταταξη προορισμων.

| Αλγοριθμος | Περιγραφη |
|------------|-----------|
| **movecomp** | Συγκρινει διαδρομες ελαχιστου κοστους που δημιουργηθηκαν χρησιμοποιωντας διαφορες συναρτησεις κοστους |
| **movecomp by polygon** | Το ιδιο αλλα χρησιμοποιωντας ενα πολυγωνο |
| **moverank** | Κατατασσει τους προορισμους κατα κοστος μετακινησης απο μια προελευση |
| **moverank by polygon** | Το ιδιο αλλα χρησιμοποιωντας ενα πολυγωνο |

### Πως να εκτελεσετε εναν αλγοριθμο

1. Επιλεξτε τον επιθυμητο αλγοριθμο απο τη λιστα
2. Η διεπαφη επεξεργασιας του QGIS ανοιγει με τις συγκεκριμενες παραμετρους του αλγοριθμου
3. Ρυθμιστε τις παραμετρους εισοδου:
   - **DTM/DEM**: Επιλεξτε το raster εδαφους
   - **Σημειο(α) προελευσης**: Επιλεξτε το επιπεδο σημειων
   - **Πολυγωνο** (αν ειναι παραλλαγη "by polygon"): Επιλεξτε την περιοχη μελετης
   - **Συναρτηση κοστους**: Επιλεξτε μεταξυ των διαθεσιμων συναρτησεων (Tobler, Minetti, κλπ.)
4. Κανε κλικ στο **Εκτελεση**
5. Τα αποτελεσματα προστιθενται αυτοματα στο εργο QGIS

Οι παραλλαγες "by polygon" καθε αλγοριθμου σας επιτρεπουν να:
- **Περιορισετε την περιοχη αναλυσης** σε μια συγκεκριμενη περιοχη
- **Μειωσετε τον χρονο υπολογισμου** εργαζομενοι με ενα περικομμενο DTM
- **Εστιασετε την αναλυση** σε μια περιοχη αρχαιολογικου ενδιαφεροντος

## Λειτουργικη ροη εργασιας

### Παραδειγμα βημα προς βημα: Υπολογισμος μιας διαδρομης ελαχιστου κοστους

Αυτο το παραδειγμα δειχνει πως υπολογιζεται μια διαδρομη ελαχιστου κοστους μεταξυ ενος οικισμου και μιας πηγης νερου.

### Βημα 1: Προετοιμασια δεδομενων

```
1. Φορτωστε το DTM της περιοχης μελετης στο εργο QGIS
2. Δημιουργηστε ενα επιπεδο σημειων με:
   - Σημειο προελευσης (οικισμος)
   - Σημειο(α) προορισμου (πηγη νερου)
3. Βεβαιωθειτε οτι το συστημα αναφορας συντεταγμενων ειναι συνεπες
```

### Βημα 2: Ελεγχος προαπαιτουμενων

```
1. Ανοιξτε το MoveCost απο τη γραμμη εργαλειων
2. Πηγαινετε στην καρτελα Ρυθμισεων
3. Κανε κλικ στο "Εγκατασταση scripts R" (αν ειναι η πρωτη φορα)
4. Βεβαιωθειτε οτι δεν αναφερονται σφαλματα
```

### Βημα 3: Εκτελεση της αναλυσης

```
1. Μεταβειτε στην καρτελα Αλγοριθμων
2. Επιλεξτε "movecost" απο την Ομαδα 1
3. Στο παραθυρο επεξεργασιας:
   - DTM: επιλεξτε το raster εδαφους
   - Προελευση: επιλεξτε το σημειο του οικισμου
   - Προορισμος: επιλεξτε το σημειο της πηγης νερου

4. Κανε κλικ στο Εκτελεση
5. Περιμενετε την ολοκληρωση της επεξεργασιας
```

## Αντιμετωπιση προβληματων

### Το R δεν βρεθηκε

**Συμπτωμα**: Μηνυμα σφαλματος "R δεν βρεθηκε" η "R is not installed"

**Λυσεις**:
1. Βεβαιωθειτε οτι το R ειναι εγκατεστημενο: ανοιξτε ενα τερματικο και πληκτρολογηστε `R --version`
2. Ελεγξτε το μονοπατι του R στις ρυθμισεις του Processing:
   - **QGIS** > **Ρυθμισεις** > **Επιλογες** > **Επεξεργασια** > **Παροχοι** > **R**
   - Οριστε το **μονοπατι φακελου R** σωστα
3. Σε macOS, το R μπορει να βρισκεται στο `/Library/Frameworks/R.framework/Resources/`
4. Σε Windows, συνηθως στο `C:\Program Files\R\R-4.x.x\`
5. Σε Linux, ελεγξτε με `which R`

### Τα scripts R λειπουν

**Συμπτωμα**: Οι αλγοριθμοι δεν εμφανιζονται στην εργαλειοθηκη επεξεργασιας

**Λυσεις**:
1. Ανοιξτε MoveCost > Ρυθμισεις > κανε κλικ στο **Εγκατασταση scripts R**
2. Επανεκκινηστε το QGIS μετα την εγκατασταση των scripts
3. Βεβαιωθειτε οτι ο Processing R Provider ειναι ενεργος:
   - **QGIS** > **Προσθετα** > **Διαχειριση και εγκατασταση προσθετων** > Ελεγξτε "Processing R Provider"
4. Ελεγξτε τον φακελο scripts R: `{QGIS_HOME}/processing/rscripts/`

### Τα γραφηματα R δεν εμφανιζονται

**Συμπτωμα**: Η καρτελα Αποτελεσματων δεν εμφανιζει κανενα γραφημα

**Λυσεις**:
1. Κανε κλικ στο **Ανανεωση** στην καρτελα Αποτελεσματων
2. Χρησιμοποιηστε **Χειροκινητη επιλογη** για να πλοηγηθειτε στον φακελο γραφηματων
3. Βεβαιωθειτε οτι η αναλυση ολοκληρωθηκε με επιτυχια
4. Ελεγξτε τους προσωρινους καταλογους:
   - macOS/Linux: `/tmp/` η `$TMPDIR`
   - Windows: `%TEMP%`
5. Μερικοι αλγοριθμοι μπορει να μη δημιουργουν γραφηματα

### Το πακετο movecost δεν ειναι εγκατεστημενο στο R

**Συμπτωμα**: Σφαλμα "there is no package called 'movecost'"

**Λυσεις**:
1. Ανοιξτε το R η RStudio
2. Εκτελεστε: `install.packages("movecost")`
3. Ελεγξτε: `library(movecost)` -- δεν πρεπει να παραγει σφαλματα
4. Σε περιπτωση προβληματων εξαρτησεων: `install.packages("movecost", dependencies = TRUE)`

### Πολυ αργη αναλυση

**Συμπτωμα**: Η επεξεργασια διαρκει πολυ χρονο

**Λυσεις**:
1. Χρησιμοποιηστε τις παραλλαγες **"by polygon"** για να περιορισετε την περιοχη υπολογισμου
2. Μειωστε την αναλυση του DTM (επαναδειγματοληψια)
3. Ελεγξτε τις διαστασεις του DTM:
   - Πολυ μεγαλα DTM (>10000x10000 pixels) απαιτουν σημαντικο χρονο
   - Περικοψτε το DTM στην περιοχη ενδιαφεροντος πριν την αναλυση
4. Κλειστε αλλες εφαρμογες για να ελευθερωσετε RAM

### Σφαλμα προβολης / CRS

**Συμπτωμα**: Ασυνεπη αποτελεσματα η σφαλμα συστηματος αναφορας συντεταγμενων

**Λυσεις**:
1. Βεβαιωθειτε οτι το DTM και τα επιπεδα σημειων εχουν το **ιδιο CRS**
2. Χρησιμοποιηστε ενα **προβαλλομενο CRS** (μετρικο), οχι γεωγραφικο
3. Συνιστωμενο CRS: UTM (π.χ. EPSG:32632 για κεντρικη Ιταλια)
4. Αναπροβαλλετε τα επιπεδα αν χρειαστει πριν την αναλυση

### Αρχιτεκτονικη του εργαλειου

Το MoveCost υλοποιειται ως αυτοτελης **QDialog** (`MoveCostDialog`) που:
- Συνεργαζεται με το QGIS Processing Framework για την εκτελεση των αλγοριθμων R
- Διαβαζει αποτελεσματα απο τα επιπεδα που ειναι φορτωμενα στο εργο
- Διαχειριζεται την απεικονιση γραφηματων R μεσω QLabel/QPixmap
- Δημιουργει αναφορες HTML χρησιμοποιωντας προκαθορισμενα προτυπα
