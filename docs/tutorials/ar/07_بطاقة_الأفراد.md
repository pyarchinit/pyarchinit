# الدرس 07: بطاقة الأفراد

## المقدمة

**بطاقة الأفراد** هي وحدة PyArchInit المخصصة للتوثيق الأنثروبولوجي للرفات البشرية المكتشفة أثناء الحفر. تسجل هذه البطاقة معلومات عن الجنس والعمر ووضعية الجسد وحالة حفظ الهيكل العظمي.

### المفاهيم الأساسية

**الفرد في PyArchInit:**
- الفرد هو مجموعة من البقايا العظمية المنسوبة إلى شخص واحد
- مرتبط ببطاقة القبر (السياق الجنائزي)
- مرتبط ببطاقة الهيكل (الهيكل المادي)
- يمكن ربطه بعلم آثار الحيوان للتحليلات المحددة

**البيانات الأنثروبولوجية:**
- تقدير الجنس البيولوجي
- تقدير العمر عند الوفاة
- وضعية واتجاه الجسد
- حالة الحفظ والاكتمال

## حقول التعريف

تحدد حقول التعريف الفرد بشكل فريد في قاعدة البيانات.

![حقول التعريف](images/07_scheda_individui/06_campi_identificativi.png)

### الموقع

**الحقل**: `comboBox_sito`
**قاعدة البيانات**: `sito`

اختيار الموقع الأثري المنتمي إليه.

### المنطقة

**الحقل**: `comboBox_area`
**قاعدة البيانات**: `area`

منطقة الحفر داخل الموقع. تُملأ القيم من القاموس.

### الوحدة الطبقية

**الحقل**: `comboBox_us`
**قاعدة البيانات**: `us`

الوحدة الطبقية المرجعية.

### رقم الفرد

**الحقل**: `lineEdit_individuo`
**قاعدة البيانات**: `nr_individuo`

الرقم التسلسلي للفرد. يُقترح تلقائياً الرقم المتاح التالي.

**ملاحظات:**
- يجب أن تكون التركيبة: الموقع + المنطقة + الوحدة الطبقية + رقم الفرد فريدة
- يُخصص الرقم تلقائياً عند الإنشاء

### ربط الهيكل

| الحقل | قاعدة البيانات | الوصف |
|-------|---------------|-------|
| رمز الهيكل | `sigla_struttura` | رمز الهيكل (مثل TM) |
| رقم الهيكل | `nr_struttura` | رقم الهيكل |

تربط هذه الحقول الفرد بالهيكل الجنائزي.

## تبويب البيانات الوصفية

يحتوي التبويب الأول على البيانات الأنثروبولوجية الأساسية.

![تبويب البيانات الوصفية](images/07_scheda_individui/08_tab_descrittivi.png)

### تقدير الجنس

**الحقل**: `comboBox_sesso`
**قاعدة البيانات**: `sesso`

تقدير الجنس البيولوجي بناءً على الخصائص المورفولوجية.

**القيم:**
| القيمة | الوصف |
|--------|-------|
| ذكر | خصائص ذكورية واضحة |
| أنثى | خصائص أنثوية واضحة |
| ذكر محتمل | غلبة الخصائص الذكورية |
| أنثى محتملة | غلبة الخصائص الأنثوية |
| غير محدد | غير قابل للتحديد |

**معايير التحديد:**
- مورفولوجية الحوض
- مورفولوجية الجمجمة
- القوة العامة للهيكل العظمي
- أبعاد العظام

### تقدير العمر عند الوفاة

| الحقل | قاعدة البيانات | الوصف |
|-------|---------------|-------|
| العمر الأدنى | `eta_min` | الحد الأدنى للتقدير |
| العمر الأقصى | `eta_max` | الحد الأقصى للتقدير |

**طرق التقدير:**
- الارتفاق العاني
- السطح الأذني
- الدروز القحفية
- التطور السني (للأحداث)
- المشاشات العظمية (للأحداث)

### فئات العمر

**الحقل**: `comboBox_classi_eta`
**قاعدة البيانات**: `classi_eta`

التصنيف حسب الفئات العمرية.

**القيم النموذجية:**
| الفئة | العمر التقريبي |
|-------|---------------|
| رضيع أول | 0-6 سنوات |
| رضيع ثاني | 7-12 سنة |
| حدث | 13-20 سنة |
| بالغ | 21-40 سنة |
| ناضج | 41-60 سنة |
| مسن | >60 سنة |

### الملاحظات

**الحقل**: `textEdit_osservazioni`
**قاعدة البيانات**: `osservazioni`

## سير العمل التشغيلي

### إنشاء فرد جديد

1. **فتح البطاقة**
   - عبر القائمة أو شريط الأدوات

2. **سجل جديد**
   - النقر على "سجل جديد"
   - يُقترح رقم الفرد تلقائياً

3. **بيانات التعريف**
   ```
   الموقع: مقبرة إيزولا ساكرا
   المنطقة: 1
   الوحدة الطبقية: 150
   رقم الفرد: 1
   رمز الهيكل: TM
   رقم الهيكل: 45
   ```

4. **بيانات البطاقة**
   ```
   التاريخ: 15/03/2024
   المُعد: م. روسي
   ```

5. **البيانات الأنثروبولوجية** (التبويب 1)
   ```
   الجنس: ذكر
   العمر الأدنى: 35
   العمر الأقصى: 45
   فئة العمر: بالغ

   الملاحظات: القامة المقدرة 170 سم.
   التهاب مفاصل قطني. تسوس متعدد.
   ```

6. **الاتجاه والوضعية** (التبويب 2)
   ```
   مكتمل: نعم
   مضطرب: لا
   متصل: نعم
   الطول: 165 سم
   الوضعية: مستلقٍ على الظهر
   الجمجمة: متجهة لليمين
   الأطراف العلوية: ممتدة على الجانبين
   الأطراف السفلية: ممتدة
   الاتجاه: شرق-غرب
   السمت: 90
   ```

7. **البقايا العظمية** (التبويب 3)
   - توثيق المناطق الموجودة

8. **الحفظ**
   - النقر على "حفظ"

### البحث عن الأفراد

1. النقر على "بحث جديد"
2. ملء معايير البحث:
   - الموقع
   - الجنس
   - فئة العمر
   - الوضعية
3. النقر على "بحث"
4. التنقل بين النتائج

- تسجيل ظروف الاستخراج

## استكشاف الأخطاء وإصلاحها

### المشكلة: رقم فرد مكرر

**السبب**: يوجد بالفعل فرد بنفس الرقم.

**الحل**:
1. التحقق من الترقيم الحالي
2. استخدام الرقم المقترح تلقائياً
3. التحقق من المنطقة والوحدة الطبقية

### المشكلة: الهيكل غير موجود

**السبب**: الهيكل غير موجود أو له رمز مختلف.

**الحل**:
1. التحقق من وجود بطاقة الهيكل
2. التحقق من الرمز والرقم
3. إنشاء الهيكل أولاً إذا لزم الأمر

### المشكلة: فئات العمر غير متاحة

**السبب**: القاموس غير مُهيأ.

**الحل**:
1. التحقق من تهيئة القاموس
2. التحقق من اللغة المحددة
3. الاتصال بالمسؤول
