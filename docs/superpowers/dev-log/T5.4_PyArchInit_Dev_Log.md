# Phase 1 (Foundation) — overview

Phase 1 closes action items AI01–AI05 from the T5.4 meeting of 4 May 2026. The
six deliverables are:

1. Vendor s3Dgraphy 0.1.30 → 0.1.40 inside `ext_libs/`.
2. New `VocabProvider` that parses the three s3Dgraphy JSON pillars directly
   (Option B per Reference Document v0.1 §4.5).
3. Refactor `pyarchinit_i18n_stratigraphic.py` as a backwards-compatible
   adapter over `VocabProvider`.
4. One-shot vocabulary alignment migration: `USVA/USVB → USVs`,
   `USVC → USVn`.
5. UUID v7 backfill on the three primary tables (`us_table`,
   `inventario_materiali_table`, `periodizzazione_table`) using a `node_uuid
   TEXT` column with a partial unique index.
6. Refactor 12 files under `modules/s3dgraphy/` so they consult `VocabProvider`
   instead of hardcoded EM 1.4 tables.

Plan structure: 8 Groups (0 + A through G), 37 tasks, ~150 steps,
≈ 30 commits expected. The release that closes Phase 1 is tagged
`phase1-foundation-5.1.0-alpha`. A rollback tag `pre-s3dgraphy-040` was
created at the start of execution; everything is recoverable.

# Group 0 — Pre-flight & rollback safety

**Date:** 6 May 2026 · **Status:** Done · **Commits:** `52b6f865`, `3ec3fb0a`

## What was built

- A git tag `pre-s3dgraphy-040` was created at HEAD `cf11f02` as the rollback
  point for the entire Phase 1. The tag is local-only at the moment; it will be
  pushed to `origin` together with the release tag in the final Group G step.
- An inventory of the legacy s3Dgraphy usage was captured under
  `.scratch/s3dgraphy-inventory.txt` (gitignored). It records every
  string-tagged `node.node_type = '<literal>'` assignment and every reference to
  the hard-coded `CRM_CLASSES` dictionary in `modules/s3dgraphy/`. The plan had
  estimated three string-literal assignments in `s3dgraphy_integration.py`; the
  actual count is one (line 113), with two further non-literal assignments
  (line 194 uses `.get(...)`) — Group F will reconcile against this evidence.
- A pytest scaffold for the upcoming `VocabProvider` test suite was created
  under `tests/sync/`. It includes a `conftest.py` exposing five fixtures and
  three sample JSON payloads modelled on the canonical s3Dgraphy filenames
  (`s3Dgraphy_node_datamodel.json`, `s3Dgraphy_connections_datamodel.json`,
  `em_visual_rules.json`) — all with no trailing space, matching the 0.1.40
  release shape.
- The repository's `.gitignore` was tightened from `/tests/` to `/tests/*` plus
  a re-include `!/tests/sync/`, so newly added test files in `tests/sync/` are
  tracked while legacy already-tracked test files in the rest of `tests/` are
  unaffected.

## What was tested

- All three JSON fixtures parse cleanly with `json.load`.
- `python3 -m pytest tests/sync/ --collect-only -q` exits cleanly with
  "no tests collected" (because no test files exist yet — they arrive in
  Group B).
- Spec compliance review (independent subagent, no shared context with the
  implementer): all nine verification checks pass; one observation about plan
  accuracy on the inventory count (non-blocking).

## Decisions taken

- The `vocab_dir` pytest fixture writes the sample JSONs into `tmp_path/JSON_config/`
  using the canonical filenames. This means downstream `VocabProvider` tests
  exercise the post-typo path; a separate test in Group B covers the legacy
  trailing-space filename of 0.1.30 explicitly so the migration scenario stays
  pinned.
- The rollback tag is annotated (not lightweight) so its message survives the
  final push; it carries the explicit sentence: *"Rollback point before
  s3Dgraphy 0.1.30→0.1.40 vendor swap and Phase 1 refactor."*

# Group A — Vendor s3Dgraphy 0.1.30 → 0.1.40

**Date:** 6 May 2026 · **Status:** Done · **Commit:** `5988ed4a`

## What was built

- The vendored s3Dgraphy was upgraded in place: `ext_libs/s3dgraphy-0.1.30.dist-info`
  removed, `pip install --target ext_libs/ --no-deps s3dgraphy==0.1.40`
  installed the new 194-file payload (3.3 MB) cleanly with no transitive
  dependencies. The new `ext_libs/s3dgraphy/__init__.py` reports
  `__version__ = "0.1.40"` and `__datamodel_version__ = "1.5.4"`.
- `requirements.txt` floor for s3Dgraphy bumped from `>=0.1.20` to `>=0.1.40`,
  which is the source-controlled record of the vendor swap. End-users who run
  `scripts/modules_installer.py` get 0.1.40 from this commit forward.
- The bundled `JSON_config/` now contains the canonical filenames
  (`s3Dgraphy_node_datamodel.json` with no trailing space — the 0.1.30 typo is
  fixed in 0.1.40). Other files present:
  `s3Dgraphy_connections_datamodel.json`, `em_visual_rules.json`,
  `em_palette_icons.json`, `em_qualia_types.json` (+ `_additions`),
  `em_extractor_types.json`, `em_document_types.json`, `new_qualia_template.json`,
  and two outdated `.ttl` files preserved by upstream for reference.
- New 0.1.40 modules confirmed present (consumed by later Groups):
  `merge/graph_merger.py` (`GraphMerger`), `exporter/graphml/graphml_patcher.py`
  (`GraphMLPatcher`), `nodes/stratigraphic_node.py` (with
  `NegativeStratigraphicUnit` and `WorkingUnit`).

## What was tested

- s3Dgraphy package imports cleanly from `ext_libs/`:
  `python3 -c "import sys; sys.path.insert(0, 'ext_libs'); import s3dgraphy;
  print(s3dgraphy.__version__)"` → `0.1.40`.
- All s3Dgraphy 0.1.40 core modules byte-compile via `py_compile`.
- Manual QGIS smoke test (US form, Harris matrix, log inspection) is deferred
  to Group G.4 — the gate that must pass before tagging the
  `phase1-foundation-5.1.0-alpha` release.

## Decisions taken

- **`ext_libs/` stays gitignored** (line 43 of `.gitignore`). PyArchInit
  follows a *runtime install* policy — the plugin's `scripts/modules_installer.py`
  pip-installs ext_libs at first launch, and `requirements.txt` is the
  authoritative version pin. The original plan called for `git add ext_libs/`
  but that would have changed long-standing repo policy and added 3.3 MB of
  vendored bytes to the repository. Group A's actual git artefact is the
  one-line `requirements.txt` bump; the 0.1.40 vendor lives only on each
  developer/user's local install. The plan was updated mid-execution to
  document this correction (the `pre-s3dgraphy-040` tag still marks the
  pre-Phase-1 commit hash but does NOT snapshot the ext_libs bytes).
- **Rollback procedure now reads:** `pip install s3dgraphy==0.1.30 --target
  ext_libs/ --no-deps`. Combined with `git checkout pre-s3dgraphy-040` for
  source-controlled state.
