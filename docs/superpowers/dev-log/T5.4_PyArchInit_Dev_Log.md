# Phase 1 (Foundation) — overview

Phase 1 closes action items AI01–AI05 from the T5.4 meeting of 4 May 2026. The
six deliverables are:

1. Vendor s3Dgraphy 0.1.30 → 0.1.40 inside `ext_libs/`.
2. New `VocabProvider` that parses the three s3Dgraphy JSON pillars directly
   (Option B per Reference Document v0.1 §4.5).
3. Refactor `pyarchinit_i18n_stratigraphic.py` as a backwards-compatible
   adapter over `VocabProvider`.
4. One-shot vocabulary alignment migration: `USVA/USVB → USVs`,
   `USVC → USVn`.
5. UUID v7 backfill on the three primary tables (`us_table`,
   `inventario_materiali_table`, `periodizzazione_table`) using a `node_uuid
   TEXT` column with a partial unique index.
6. Refactor 12 files under `modules/s3dgraphy/` so they consult `VocabProvider`
   instead of hardcoded EM 1.4 tables.

Plan structure: 8 Groups (0 + A through G), 37 tasks, ~150 steps,
≈ 30 commits expected. The release that closes Phase 1 is tagged
`phase1-foundation-5.1.0-alpha`. A rollback tag `pre-s3dgraphy-040` was
created at the start of execution; everything is recoverable.

  (`GraphMLPatcher`), `nodes/stratigraphic_node.py` (with

## What was tested

- s3Dgraphy package imports cleanly from `ext_libs/`:
  `python3 -c "import sys; sys.path.insert(0, 'ext_libs'); import s3dgraphy;
  print(s3dgraphy.__version__)"` → `0.1.40`.
- All s3Dgraphy 0.1.40 core modules byte-compile via `py_compile`.
- Manual QGIS smoke test (US form, Harris matrix, log inspection) is deferred
  to Group G.4 — the gate that must pass before tagging the
  `phase1-foundation-5.1.0-alpha` release.

## Decisions taken

- **`ext_libs/` stays gitignored** (line 43 of `.gitignore`). PyArchInit
  follows a *runtime install* policy — the plugin's `scripts/modules_installer.py`
  pip-installs ext_libs at first launch, and `requirements.txt` is the
  authoritative version pin. The original plan called for `git add ext_libs/`
  but that would have changed long-standing repo policy and added 3.3 MB of
  vendored bytes to the repository. Group A's actual git artefact is the
  one-line `requirements.txt` bump; the 0.1.40 vendor lives only on each
  developer/user's local install. The plan was updated mid-execution to
  document this correction (the `pre-s3dgraphy-040` tag still marks the
  pre-Phase-1 commit hash but does NOT snapshot the ext_libs bytes).
- **Rollback procedure now reads:** `pip install s3dgraphy==0.1.30 --target
  ext_libs/ --no-deps`. Combined with `git checkout pre-s3dgraphy-040` for
  source-controlled state.

# Group B — VocabProvider (Option B per-tool JSON parsing)

**Date:** 6–7 May 2026 · **Status:** Done · **Commits:** 11 (`02d59b07` → `88e70498`)

## What was built

The `modules/s3dgraphy/sync/` package was created from scratch. It is the
PyArchInit-side adapter that consumes s3Dgraphy's three JSON pillars
directly, per the Option B preference declared in Reference Document v0.1
§4.5 (per-tool parsing rather than wrapping a typed s3Dgraphy Python API).
Every consuming surface in PyArchInit — combo boxes, validation hints,
visual rendering — will read from this single provider in subsequent
Groups; the pure-Python core can be unit-tested without QGIS, while the
Qt-aware wrapper integrates with the QGIS event loop only when needed.

The package layout:

| File | Responsibility |
|---|---|
| `modules/s3dgraphy/sync/__init__.py` | Public API surface; re-exports the typed dataclasses and the pure-Python core; provides a lazy accessor `get_vocab_provider()` for the Qt wrapper so non-Qt callers (tests, CLI scripts) never pull in PyQt at import time. |
| `modules/s3dgraphy/sync/vocab_types.py` | Five frozen dataclasses (`UnitType`, `EdgeType`, `VisualRule`, `ParadataType`, `VocabularyVersion`) plus a `Family(str, Enum)` whose values mirror the JSON top-level keys (`stratigraphic_nodes`, `paradata_nodes`, `group_nodes`, …). |
| `modules/s3dgraphy/sync/vocab_provider_core.py` | `VocabProviderCore` — pure-Python loader/queryer. Reads the three pillars from `bundled_dir`, optionally merges overrides from `overrides_dir` per top-level key, and exposes the query API: `get_unit_types(family=...)`, `get_edge_types()`, `get_legal_targets_for(...)`, `get_paradata_types()`, `get_visual_rule(...)`, `get_cidoc_mapping(...)`, `versions` property, optional `min_versions={...}` enforcement at construction time. Accepts the legacy 0.1.30 trailing-space filename `s3Dgraphy_node_datamodel .json` as a fallback. |
| `modules/s3dgraphy/sync/vocab_provider.py` | `VocabProvider` — thin Qt wrapper. Subclasses `QObject` when QGIS is importable, falls back to plain `object` otherwise. Adds a `vocabulary_changed` `pyqtSignal` and a `QFileSystemWatcher` covering both the bundled JSON directory and the user override directory, so combo-box dialogs can subscribe to changes and refresh on the fly. Defaults the bundled directory to `<plugin>/ext_libs/s3dgraphy/JSON_config/` and the overrides directory to `~/.config/pyarchinit/vocab_overrides/`. |

## What was tested

The TDD discipline laid out in plan §B was followed strictly. Every method
on `VocabProviderCore` was preceded by a failing test, then implemented,
then verified green before commit. The final test count under
`tests/sync/`:

- **17 fixture-driven tests** in `test_vocab_types.py` and
  `test_vocab_provider_core.py` covering typed dataclass shape, JSON
  loading, family filtering, edge-type loading, paradata loading, visual
  rules, CIDOC lookup, version exposure, minimum-version gating (pass +
  raise), legacy trailing-space filename fallback, and override merge
  semantics (a partial paradata override must not erase bundled
  stratigraphic types).
- **2 smoke tests** in `test_vocab_provider_smoke.py` running against the
  real `ext_libs/s3dgraphy/JSON_config/` 0.1.40 vendor installed in
  Group A. The smoke tests confirm `US`, `USVs`, `USVn` appear in the
  stratigraphic family on the actual vendored datamodel and that the
  per-pillar version field is populated. Auto-skipped when `ext_libs/`
  is missing (fresh clone before `modules_installer.py` runs).

Total: **19 passed in 0.20 s**, run via `pytest tests/sync/ -v` from the
plugin root. The runner is configured by a minimal `pyproject.toml`
([`tool.pytest.ini_options`] with `testpaths = ["tests/sync"]` and
`--import-mode=importlib --confcutdir=tests/sync`) so the new suite can
run from the CLI without QGIS bootstrap — needed because the plugin's
top-level `__init__.py` imports `qgis.PyQt.*` at module load.

Two-stage review was applied to the foundation pair (Tasks B.1 + B.2): an
independent spec-compliance reviewer confirmed byte-level match to the
plan literal source, and a code-quality reviewer approved with minor
follow-ups, all four of which were folded into the next subagent batch
without requiring a separate fix commit.

## Decisions taken

- **Pure core / Qt wrapper split** is locked in. The pure-Python
  `VocabProviderCore` is the only surface tests exercise; the Qt
  `VocabProvider` is a thin singleton-with-signal that any QGIS dialog
  can subscribe to without inheriting test infrastructure. This keeps
  the test suite QGIS-free and runnable in CI / from a bare shell.
- **`pyproject.toml` is the canonical pytest config location** for this
  plugin, scoped narrowly to `tests/sync`. Legacy tests under
  `tests/test_*.py` already required QGIS to run, so no change in
  behaviour for them; the new suite is additive.
- **The 0.1.30 trailing-space filename** (`s3Dgraphy_node_datamodel .json`
  with a literal space before the extension) is preserved as a fallback
  in `_NODE_DATAMODEL_NAMES`. It will only be removed when 0.1.30
  support is officially dropped — annotated inline so future
  maintainers find the answer without archaeology.
- **Override priority is per top-level key, not whole-file**, locked in
  by a regression test that builds a partial paradata-only override and
  asserts the bundled stratigraphic types still surface from
  `get_unit_types()`. A datacenter pulling a partial vocabulary update
  will not silently erase locally available types.
- **Public API frozen for Phase 1**: `VocabProviderCore`, `UnitType`,
  `EdgeType`, `VisualRule`, `ParadataType`, `VocabularyVersion`,
  `Family`, `get_vocab_provider()`. `SyncEngine`, `GraphProjector`,
  `ParadataStore`, `ConflictResolver` are deliberately absent — they
  belong to Phases 2 and 3.

## Plan corrections folded in mid-execution

- **`pyproject.toml` was not in the plan** but is required so the new
  test suite can run from a bare CLI. The deviation was approved during
  spec review of B.1 + B.2; the file is 6 lines, single-section, and
  scoped to `tests/sync` so legacy tests are unaffected.
- **`EdgeType.allowed_pairs` annotation tightened** from raw `tuple` to
  `tuple[tuple[str, str], ...]` (review follow-up), preserving the
  hashability contract of frozen dataclasses.
