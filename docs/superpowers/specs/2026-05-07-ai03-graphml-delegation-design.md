# AI03 — Delegate GraphML serialisation to s3Dgraphy

**Spec date:** 2026-05-07
**Author:** Enzo Cocca
**Status:** Design proposal — brainstormed and approved (sections 1–5)
**Target plugin version:** 5.2.0-alpha
**Target s3Dgraphy version:** 0.1.40 (already vendored in Phase 1)
**Action item:** **AI03** of T5.4 meeting minutes 2026-05-04, also covered by §7.6 of the parent spec
**Phase:** Phase 2 (Local graph layer), AI03 standalone — AI04 and AI05 follow as separate releases

## 1. Goal

Replace pyarchinit's broken DOT → GraphML pipeline (which is the root cause of
the four Extended Matrix limitations carried over from Phase 1) with a clean
delegation to `s3dgraphy.exporter.graphml.GraphMLExporter`, which already
ships transitive reduction, period swimlanes, and the full EM 1.5 edge-type
vocabulary. Cut-over: the legacy `dottoxml`-based code path is removed in
the same release; rollback exists via the `pre-s3dgraphy-040` tag and the
`phase1-foundation-5.1.0-alpha` release tag.

## 2. Decisions captured during brainstorming

| # | Decision | Value |
|---|---|---|
| D1 | Refactor scope | **B — clean cut-over** (no opt-in fallback, no parallel writer flag). Legacy GraphML code path is removed in this release |
| D2 | Validation strategy | **B — fixture-based pytest suite** (4 tests mapped 1:1 to the four EM limitations) **plus** manual smoke gate before tagging |
| D3 | Phase 2 sub-bundling | **A — AI03 alone** as `5.2.0-alpha`. AI04 (`populate_graph`/`populate_list`) and AI05 (`ParadataStore`) ship as separate later releases per their own brainstorming + plan |
| D4 | DOT/JSON output paths | Unchanged — the legacy `S3DGraphyIntegration.import_from_pyarchinit()` keeps producing the `Graph` consumed by DOT and JSON exporters. Only the GraphML branch is rerouted |
| D5 | "Configure spatial/functional groupings" UI | Removed from the dialog. yEd has native grouping post-import; the broken legacy enhancer is deleted |
| D6 | "Bake auxiliary data" UI toggle | Deferred to AI05. AI03 always uses `persist_auxiliary=False` (volatile mode). Adding a checkbox without effect today is misleading |
| D7 | DB-path resolution | New `db_manager.get_sqlite_path() -> Path | None`. PostgreSQL returns `None`, GraphML branch logs an Info message and skips, DOT/JSON still produced. PostgreSQL GraphML support is AI04 |
| D8 | Site filter strategy | Run `PyArchInitImporter` on the whole DB, then filter the resulting `Graph` in memory by `sito`. Volterra-grade datasets (~1500 US) make this trivial |
| D9 | Mapping ownership | `pyarchinit_us_mapping.json` lives in s3dgraphy and is co-evolved by CNR-ISPC per Reference Document v0.1 §7. AI03 does NOT modify it; column gaps surface as upstream JSON pull requests |

## 3. Architecture

### 3.1 Current path (legacy, broken)

```
PyArchInit DB
  ↓
S3DGraphyIntegration.import_from_pyarchinit()
  └─ produces: Graph with generic Node instances, no EpochNode
  ↓
s3dgraphy_to_dot()                  → file.dot         ✅ works
  ↓
_convert_dot_to_graphml()           → parses DOT
  ↓
dottoxml.exportGraphml()            ❌ pops a QInputDialog (interactive)
                                    ❌ string-naive parser pre-09b9fbde lost edges
  ↓
enhance_graphml_with_groups()       ❌ truncates the file when post-hoc
                                       grouping fails (yfiles namespace mismatch)
  ↓
spatial_grouping_manager            ❌ apply_grouping_to_dot drops nodes
                                       not assigned to any group
```

### 3.2 New path (AI03 cut-over)

```
PyArchInit DB
  ├──► S3DGraphyIntegration.import_from_pyarchinit() (UNCHANGED)
  │    └─ feeds DOT and JSON exports — both already work, kept verbatim
  │
  └──► (NEW) modules.s3dgraphy.sync.graphml_writer.export_graphml()
       │
       ├─ s3dgraphy.PyArchInitImporter(db_path, mapping='pyarchinit').parse()
       │  └─ produces: Graph with typed StratigraphicUnit / USVs / USVn /
       │     EpochNode instances, has_first_epoch edges generated by
       │     base_importer.py:577–651
       │
       ├─ _filter_by_site(graph, site)  (in-memory)
       │
       └─ s3dgraphy.exporter.graphml.GraphMLExporter(graph)
                       .export(output_path, persist_auxiliary=False)
            ├─ aux_tracking volatile policy
            ├─ TemporalInferenceEngine.transitive_reduction()
            │  closes limitation L4 ("no tred")
            ├─ EpochSwimlanesGenerator emits TableNode swimlane
            │  closes limitation L2 ("no period rows")
            ├─ EdgeGenerator with full EM 1.5 vocabulary (36 edge types)
            │  closes limitation L3 ("partial edge styling")
            └─ XML written; closes limitation L1 ("empty graphml on grouping flag")
```

### 3.3 Code removed in this release

- `modules/s3dgraphy/s3dgraphy_dot_bridge.py::S3DGraphyDotBridge._convert_dot_to_graphml()` — superseded
- `modules/s3dgraphy/s3dgraphy_dot_bridge.py::S3DGraphyDotBridge` — internal `Options` class for dottoxml — superseded
- `modules/s3dgraphy/graphml_spatial_enhancer.py` — entire file deleted (replaced by `EpochSwimlanesGenerator`)
- `modules/s3dgraphy/spatial_grouping_manager.py::SpatialGroupingManager.apply_grouping_to_dot()` — deleted
- `modules/s3dgraphy/spatial_grouping_manager.py::SpatialGroupingDialog` — deleted (along with the `cb_spatial_grouping` checkbox in `S3DGraphyExportDialog`)
- imports of `dottoxml` and `dot` from `s3dgraphy_dot_bridge.py` — deleted

`resources/dbfiles/dot.py` and `resources/dbfiles/dottoxml.py` are NOT deleted: other legacy code paths (Harris matrix exporters in `tabs/Interactive_matrix.py`, `modules/utility/pyarchinit_matrix_exp.py`) still call them. Their removal is scheduled for AI04+ when those exporters also delegate to s3dgraphy.

## 5. Data flow

The dialog flow at the green "Extended Matrix" button:

1. User opens the dialog, default checkboxes: DOT ✓, GraphML ✓, JSON ✓, Phased ☐. Clicks Export.
2. `S3DGraphyDotBridge.export_integrated_matrix(site, area, output_dir, formats)` orchestrates per-format export. Each format is independent (failure of one does not block the others).
3. **DOT branch**: `s3dgraphy_to_dot(site, area)` → `<base>.dot`. Unchanged from Phase 1.
4. **JSON branch**: `s3d_integration.export_to_json(json_path)` → `<base>_s3dgraphy.json`. Unchanged from Phase 1.
5. **Phased branch**: `s3d_integration.export_phased_matrix(phased_path)` → `<base>_phased.json`. Unchanged from Phase 1.
6. **GraphML branch (NEW)**:
   1. Resolve `db_path = self.db_manager.get_sqlite_path()`.
   2. If `db_path is None` (PostgreSQL): log Info, skip GraphML, mark per_format_status as "skipped: PG support in AI04". Continue to step 7.
   3. Call `graphml_writer.export_graphml(db_path, 'pyarchinit', graphml_path, site_filter=site, persist_auxiliary=False)`.
   4. Inside `export_graphml`:
      1. `PyArchInitImporter(filepath=db_path, mapping_name='pyarchinit').parse()` → typed `Graph`.
      2. If `site_filter`: `_filter_by_site(graph, site_filter)` returns a filtered `Graph` whose `nodes` keep all `StratigraphicNode` subclass instances whose `sito` attribute matches `site_filter`, **plus** every `EpochNode` reachable via a `has_first_epoch` edge from any retained stratigraphic node. Edges between two retained nodes are kept; edges with at least one endpoint dropped are pruned. EpochNodes have no `sito` of their own — they are global vocabulary objects shared across sites; reachability is the only sound retention rule.
      3. If `len(graph.nodes) == 0`: raise `EmptyGraphError`.
      4. `GraphMLExporter(graph).export(output_path, persist_auxiliary=False)` writes XML.
      5. Build and return `ExportResult(output_path, node_count, edge_count, epoch_count, tred_removed_edges, warnings)`.
   5. Bridge stores the `ExportResult` (or the error) in `per_format_status['graphml']`.
7. **Final summary dialog**: `QMessageBox.information` with one line per format describing what was produced (path, metrics) or what failed (error class + reason). Format the GraphML line as e.g. *"GraphML exported: 298 nodes, 5 epochs, 655 → 412 edges (243 redundancies removed by transitive reduction), 0 warnings."*

`GraphMLExporter`'s internal `print()` calls are routed to `QgsMessageLog` via a context manager so the user sees per-stage progress in the QGIS log panel without the bridge having to add its own progress bar.

## 6. Error handling

Every export path is independent — a GraphML failure does NOT block DOT/JSON. The bridge catches and categorises every failure mode below; the user always sees a final summary dialog telling them which formats succeeded and why the others did not.

| Stage | Error | Trigger | Treatment |
|---|---|---|---|
| Pre-flight | DB is PostgreSQL | `db_manager.get_sqlite_path()` returns `None` | Log Info, skip only GraphML, summary dialog notes "PG support in AI04". DOT/JSON still produced |
| Pre-flight | DB path missing | `FileNotFoundError` from importer | Log Warning with path, summary dialog "GraphML failed: DB not found", DOT/JSON not touched |
| Pre-flight | `lxml` not installed | `ImportError` on `from lxml import etree` (transitively required by `GraphMLExporter`) | Log Critical pointing user to `python scripts/modules_installer.py`, skip GraphML |
| Mapping | `pyarchinit_us_mapping.json` not found | s3dgraphy raises `MappingNotFoundError` | Log Warning, summary "GraphML mapping missing — please reinstall plugin" |
| Mapping | Schema mismatch (DB lacks a column the mapping expects) | s3dgraphy logs warnings and continues with available fields | Pyarchinit captures s3dgraphy logger output and surfaces in `ExportResult.warnings` |
| Filter | `site_filter` matches zero records | `len(graph.nodes) == 0` after filter | Raise `EmptyGraphError`, summary "No US records for site '<X>'" |
| Temporal | Cycle in stratigraphic relations | `TemporalInferenceEngine.transitive_reduction()` raises `ValueError("cycle detected: ...")`, already handled at `graphml_exporter.py:117–119` | s3dgraphy adds the warning to `graph.warnings` and falls back to non-reduced edges. Pyarchinit propagates the warning through `ExportResult.warnings` so the user sees it in the summary dialog |
| Filesystem | Permission denied / disk full | `OSError` from `tree.write()` | Wrap in `GraphMLExportError('write', original)`, log Critical, summary "Failed to write GraphML to <path>" |
| Internal s3dgraphy bug | Anything else from `GraphMLExporter` | `Exception` | Wrap in `GraphMLExportError('export', original)`, log full traceback in QGIS log panel for bug reports, summary "GraphML export failed: <type>: <msg>" with a "Show details" button that opens the log panel |

The exception hierarchy:

```python
class EmptyGraphError(ValueError):
    """Graph has no nodes after import + filter."""

class GraphMLExportError(RuntimeError):
    """Wraps any failure during the GraphML export pipeline."""
    def __init__(self, stage: str, original: Exception):
        # stage ∈ {'import', 'filter', 'export', 'write'}
        self.stage = stage
        self.original = original
        super().__init__(
            f"GraphML {stage} failed: {type(original).__name__}: {original}")
```

## 7. Testing strategy

### 7.1 Levels

| Level | Tests | Where it runs | When it runs |
|---|---|---|---|
| L1 — Unit | 3 tests on helpers (`ExportResult`, `_filter_by_site`, `GraphMLExportError(stage=...)`) | `pytest tests/sync/` (no QGIS) | Every commit |
| L2 — Fixture-based pipeline | 4 tests mapped 1:1 to the four closed limitations | `pytest tests/sync/` (no QGIS — `GraphMLExporter` does not import Qt) | Every commit |
| L3 — Manual smoke | Dialog in QGIS, click green button, verify yEd output | Manually in QGIS on the Volterra/Castelseprio dataset | Pre-tag of the `5.2.0-alpha` release |

### 7.2 The four L2 tests (acceptance criteria)

1. **`test_pipeline_produces_populated_graphml`** (L1 of dev-log): asserts `out.exists()`, `out.stat().st_size > 1000`, `result.node_count > 0`, `result.edge_count > 0`.
2. **`test_pipeline_emits_epoch_swimlanes`** (L2): asserts the produced XML contains `TableNode` or `yfiles.foldertype="row"` markers and `result.epoch_count >= 2` (the fixture has two periods).
3. **`test_pipeline_diversifies_edge_styles`** (L3): asserts the XML contains at least 2 distinct `<y:LineStyle ... type="..."/>` values, given that the fixture includes `copre` (cuts), `uguale a` (has_same_time) and `riempie` (fills).
4. **`test_pipeline_applies_transitive_reduction`** (L4): asserts `result.tred_removed_edges >= 1`, given that the fixture includes one explicit transitive redundancy (`US1 → US2 → US3` plus `US1 → US3`).

### 7.3 The fixture

`tests/sync/fixtures/mini_volterra.sqlite` is generated once by
`tests/sync/fixtures/build_mini_volterra.py` and committed to the repo
(~80–150 KB binary). The script copies the schema from
`resources/dbfiles/pyarchinit.sqlite` (the canonical template) and inserts
hand-crafted rows that exercise:

- A single `sito = 'TestSite'` so the L2 test that calls `export_graphml` without a `site_filter` and the variant that filters by `sito='TestSite'` both produce the same graph (a regression on `_filter_by_site` that drops everything would surface immediately).
- 5 US: 3 with `unita_tipo='US'`, 1 with `unita_tipo='USM'`, 1 with `unita_tipo='USVs'`.
- 2 periods/phases in `periodizzazione_table` (e.g. *Late Roman*, *Early Medieval*), with at least one US assigned to each period via `periodo_iniziale`/`fase_iniziale`. This ensures both `EpochNode`s survive the site filter through `has_first_epoch` edges and `result.epoch_count >= 2` holds.
- 7 `rapporti` entries spanning 4 distinct `rel_type` values (`copre`, `coperto da`, `uguale a`, `riempie`).
- 1 deliberate transitive redundancy that the L4 test pins.

Regenerating the fixture is a deliberate, manual step (`python tests/sync/fixtures/build_mini_volterra.py`); commits update the binary alongside the script change.

### 7.4 Manual smoke gate

Pre-tag for `phase2-ai03-graphml-delegation-5.2.0-alpha`:

1. Restart QGIS, open the Volterra/Castelseprio project.
2. Click the green "Extended Matrix" button → Export with default checkboxes.
3. Open the produced `.graphml` in yEd:
   - period swimlanes visible as horizontal rows
   - edge types visually distinct (solid for stratigraphic, dashed for `has_same_time`, etc.)
   - edge count visibly lower than DOT count (transitive reduction at work)
4. Verify the final `QMessageBox` reports the metrics correctly (node/edge/epoch/tred-removed counts).
5. Edge case: enter a non-existent site filter; verify the summary dialog shows `EmptyGraphError` cleanly without blocking DOT/JSON.
6. Regression check: confirm `.dot` and `_s3dgraphy.json` still produced and unchanged from Phase 1.

If any of the six checks fails: roll back to `phase1-foundation-5.1.0-alpha` and investigate before re-tagging.

## 8. Acceptance criteria

The release `5.2.0-alpha` ships when **all** of the following hold:

- All 19 Phase 1 tests still pass (`pytest tests/sync/ tests/migrations/`).
- All 7 new tests added by AI03 pass (3 unit + 4 fixture-pipeline).
- Manual smoke gate (§7.4) passes on the Volterra dataset.
- The four EM limitations from the Phase 1 dev-log are demonstrably closed:
  - L1 — non-empty `.graphml` on every export path.
  - L2 — `EpochNode`s emitted as TableNode rows.
  - L3 — at least 2 distinct yEd `LineStyle.type` values present in real exports.
  - L4 — `tred_removed_edges > 0` on real exports with redundancies.
- `metadata.txt` bumped to `5.2.0-alpha` with a bilingual changelog entry.
- The Phase 2 dev-log section in `T5.4_PyArchInit_Dev_Log.md` summarises the
  closure and is rendered to DOCX for Teams.

## 9. Out of scope

The following are deliberately deferred and are NOT part of AI03:

- **PostgreSQL-backed GraphML export.** AI03 logs Info and skips the GraphML branch when the backend is PG. AI04 (`GraphProjector` proper) extends `s3dgraphy.PyArchInitImporter` or wraps a SQLAlchemy-driven importer to cover the PG case.
- **Bidirectional bridge** (`populate_graph` / `populate_list`). AI04.
- **Paradata management** (authors, licenses, virtual reconstructions, embargo). AI05 with the `ParadataStore`.
- **Custom spatial/functional groupings** as yEd group nodes inside the swimlane. Removed in AI03; if there is partner demand later we re-introduce it as a post-export enhancement layer.
- **GraphML for the Harris-matrix exporter** in `tabs/Interactive_matrix.py` and `modules/utility/pyarchinit_matrix_exp.py`. They still use the legacy `dot.py`/`dottoxml.py` runtime files in `~/pyarchinit/bin/`. Migrating them is its own follow-up under AI04 or later.
- **Networking / SyncEngine / DatacenterClient.** Phase 3.
