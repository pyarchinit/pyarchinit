# PG-UIFix — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hotfix milestone removing 5 obsolete SQLite-only guards in `gui/dialog_paradata_manager.py` and `modules/s3dgraphy/s3dgraphy_dot_bridge.py` that block PostgreSQL backend users from using Paradata Manager + GraphML import/export, even though the underlying machinery shipped with Phase 3 PG-Compat (Foundation + PG-A through PG-D, 2026-05-10/11) supports both backends.

**Architecture:** Pure mechanical removal of dead code paths. The `_resolve_db_handle` shim (PG-D era) in `ParadataStore.__init__`, `GroupStore.__init__`, and `GraphIngestor.populate_list()` already accept `Path | DbHandle | str` for `db_path`. The plan removes 5 SQLite-only guards (3 in dialog, 2 in bridge) and 1 `if backend_is_postgres: skip` graphml-export branch. One spot (US picker, line 414) requires a small SQLAlchemy rewrite because the existing code uses raw `sqlite3.connect()` instead of the db_manager engine.

**Tech Stack:** Python 3.9+, SQLAlchemy 2.0+ (for US picker rewrite), pytest with `unittest.mock`, no Qt rendering needed in tests, no new dependencies.

**Spec source of truth:** `docs/superpowers/specs/2026-05-12-pg-uifix-design.md` (commit `ef4c35dd`).

**Predecessor tag:** `yed-import-parsers-5.7.7-alpha` (commit `5d666c67`).

**Target tag:** `pg-uifix-5.7.8-alpha`.
